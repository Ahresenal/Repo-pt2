📉 Predicción de Evasión de Clientes - Telecom X🎯 Objetivo del ProyectoTras identificar los patrones de comportamiento en la Fase 1, esta segunda etapa se enfoca en la Ciencia de Datos Aplicada. El objetivo es construir, entrenar y validar un modelo de Machine Learning capaz de predecir qué clientes tienen mayor probabilidad de abandonar la empresa (Churn), permitiendo a Telecom X tomar decisiones preventivas.🛠️ Metodología y Flujo de Trabajo1. Preparación de Datos (ETL & Wrangling)Consistencia: Se utilizó el dataset tratado proveniente de la Fase 1 (limpieza de JSON y normalización de nombres).Ingeniería de Características: Se eliminaron variables irrelevantes como customerID y se transformaron las variables categóricas a numéricas mediante One-Hot Encoding.Variable Objetivo: Se codificó la columna Churn como binaria ($1$ para "Yes", $0$ para "No").2. Desarrollo del ModeloSe implementó un algoritmo de Regresión Logística, ideal para problemas de clasificación binaria.División de datos: 70% para entrenamiento y 30% para pruebas (Train/Test Split).Validación: El modelo fue evaluado con una matriz de confusión para medir su capacidad real de detección.📊 Resultados y Métricas FinalesEl modelo demostró ser robusto y confiable para la toma de decisiones:Exactitud (Accuracy): 81%. El modelo acierta en 8 de cada 10 predicciones.Matriz de Confusión:Verdaderos Positivos: Se identificaron con éxito 305 clientes en riesgo inminente de fuga.Verdaderos Negativos: Se clasificaron correctamente 1413 clientes leales.💡 Conclusiones EstratégicasBasado en las predicciones, se recomienda a Telecom X:Acción Inmediata: Focalizar campañas de fidelización en el grupo de 305 clientes detectados por el modelo.Revisión de Contratos: Incentivar la transición de contratos mensuales a anuales, ya que el modelo identifica el corto plazo como el factor de riesgo número uno.Optimización de Costos: Monitorear a los clientes con cargos mensuales altos, ofreciendo servicios de valor agregado (como Soporte Técnico) para aumentar la retención.📂 Archivos en este RepositorioPrediccion_Evasion_TelecomX.ipynb: Notebook con todo el pipeline de Machine Learning.datos_tratados_telecom.csv: Dataset final utilizado para el entrenamiento del modelo.
